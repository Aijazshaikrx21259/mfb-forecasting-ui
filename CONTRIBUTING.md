# Contributing to MFB Forecasting UI

## Welcome!

Thank you for considering contributing to the MFB Forecasting UI project.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Copy `.env.example` to `.env.local`
5. Start dev server: `npm run dev`

## Code Style

- Use TypeScript for all new code
- Follow existing patterns and conventions
- Use Tailwind CSS for styling
- Leverage shadcn/ui components
- Write meaningful commit messages

## Commit Messages

Follow conventional commits format:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting)
- `refactor:` - Code refactoring
- `test:` - Test additions or changes
- `chore:` - Maintenance tasks

Example: `feat: add backtest dashboard with performance metrics`

## Pull Request Process

1. Create a feature branch
2. Make your changes
3. Test thoroughly
4. Update documentation
5. Submit PR with clear description
6. Address review feedback

## Code Review

All PRs require review. Reviewers check:

- Code quality and style
- TypeScript types
- Component structure
- Performance
- Accessibility
- Documentation

## Testing

- Test components manually
- Verify responsive design
- Check accessibility
- Test with/without Clerk auth

## Questions?

Open an issue or contact maintainers.
